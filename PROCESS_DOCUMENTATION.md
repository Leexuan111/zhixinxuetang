### 1.2 WeChat Mini Program 配置文件

#### app.json

全局应用配置，定义：
- **pages**: 注册的三个页面路径
  - `pages/index/index` - 首页（拍照识题）
  - `pages/history/history` - 历史记录（错题本）
  - `pages/settings/settings` - 设置页
- **window**: 导航栏样式（白色文字、蓝色背景 #4A90D9）
- **tabBar**: 底部导航栏配置，包含三个选项卡
  - 拍照识题（相机图标）
  - 错题本（书本图标）
  - 设置（齿轮图标）
- **lazyCodeLoading**: 按需加载组件

#### project.config.json

微信开发者工具项目配置，包含：
- 项目名称和描述
- 编译设置
- 调试配置

### 1.3 命名规范

- **页面目录**: 使用小写英文（index, history, settings）
- **页面文件**: 四件套（.ts, .wxml, .wxss, .json）与目录同名
- **工具文件**: 存放在 `utils/` 目录，使用小驼峰命名
- **资源文件**: 存放在 `images/` 目录，使用 kebab-case 命名
- **类型定义**: 使用 TypeScript 接口（I前缀，如 `IHistoryRecord`）

## 2. 拍照识题模块

### 2.1 页面功能概述

**页面**: `pages/index/index`
**名称**: 拍照识题 / 首页

**核心功能**:
1. 拍照或从相册选择错题图片
2. 调用AI分析错题内容
3. 显示识别结果（原题、错因分析、举一反三练习题）
4. 自动保存到错题本

**页面状态**:
- **初始状态**: 显示上传按钮，提示用户拍照或选择图片
- **选择图片后**: 显示图片预览，可进行重新选择或开始分析
- **分析中**: 显示加载动画，提示"AI正在分析错题"
- **结果展示**: 显示识别的题目、错因分析、举一反三练习题

### 2.2 用户流程：进入拍照识题

```
启动小程序
    ↓
默认进入【首页/拍照识题】
    ↓
显示拍照/选择图片入口
```

**入口点**:
- 小程序启动时默认进入首页
- 从底部Tab栏点击"拍照识题"

**导航触发**:
- Tab栏切换：用户点击底部"拍照识题"选项卡
- 图标状态：默认显示 `camera.png`，选中显示 `camera-active.png`

### 2.3 用户流程：拍照/选择图片

```
点击"拍照 / 选择图片"按钮
    ↓
弹出选择菜单
    ├─→ 选项1: 拍照识别（调用相机）
    └─→ 选项2: 从相册选择（调用相册）
            ↓
    用户完成拍摄/选择
            ↓
    显示图片预览
            ↓
    可选择：重新选择 / 开始分析
```

**操作选项**:
1. **拍照识别**: 调用 `wx.chooseMedia` 使用相机拍摄
2. **从相册选择**: 调用 `wx.chooseMedia` 从相册选取
3. **重新选择**: 回到选择菜单
4. **重新分析**: 使用当前图片重新分析（保留图片，清空结果）

**技术说明**:
- 使用 `wx.chooseMedia` API
- 限制选择1张图片
- 使用压缩图片 (`sizeType: ['compressed']`)
- 支持格式: jpg, jpeg, png, bmp, webp

### 2.4 用户流程：查看识别结果

```
AI分析完成
    ↓
显示结果页面
    ├─→ 【识别的题目】区域
    │       └─→ 显示原题内容 + 学科标签 + 知识点标签
    ├─→ 【错因分析】区域（如存在）
    │       └─→ 显示AI分析的可能错误原因
    └─→ 【举一反三·练习题】区域
            ├─→ 练习题1（可展开答案）
            ├─→ 练习题2（可展开答案）
            └─→ 练习题3（可展开答案）
                    ↓
    底部操作按钮
        ├─→ 📸 继续拍照（重置页面）
        └─→ 🔄 重新分析（保留图片，重新分析）
```

**结果区域详解**:

| 区域 | 内容 | 交互 |
|------|------|------|
| 识别的题目 | 原题文本、学科标签、知识点标签 | 文本展示，点击查看大图 |
| 错因分析 | AI分析的出错原因 | 文本展示 |
| 举一反三练习题 | 3道相似题目 | 每道题可展开/收起答案 |

**交互说明**:
- **查看答案**: 点击"查看答案/收起答案"展开或收起答案
- **预览图片**: 点击图片可全屏预览
- **继续拍照**: 清空当前状态，返回初始页面
- **重新分析**: 使用当前图片重新调用AI分析

### 2.5 导航选项分析后

**分析完成后的操作**:

1. **继续拍照** (`📸 继续拍照`)
   - 重置整个页面状态
   - 清空图片、结果、分析状态
   - 返回初始上传页面

2. **重新分析** (`🔄 重新分析`)
   - 保留当前图片
   - 清空结果展示
   - 重新调用AI分析

3. **分享** (右上角分享按钮)
   - 分享小程序给好友
   - 分享标题: "知错学堂 - 拍照识错题，举一反三"

4. **Tab栏切换**
   - 可切换到"错题本"查看刚保存的记录
   - 可切换到"设置"调整API配置

## 3. 错题本模块

### 3.1 页面功能概述

**页面**: `pages/history/history`
**名称**: 错题本

**核心功能**:
1. 展示所有历史错题记录
2. 支持展开/收起查看详情
3. 支持删除单条记录
4. 支持清空全部记录
5. 显示记录总数

**页面状态**:
- **空状态**: 显示"暂无错题记录"提示，引导用户去首页拍照
- **列表状态**: 显示错题卡片列表，每条记录包含时间、学科、摘要
- **展开状态**: 展开单条记录显示完整详情（错因分析、知识点、举一反三）

### 3.2 用户流程：访问错题本

```
点击底部Tab栏【错题本】
            ↓
    加载本地存储的历史记录
            ↓
    ├─→ 如有记录：显示列表
    │       └─→ 显示记录总数
    │       └─→ 每条记录显示时间、学科、题目摘要
    └─→ 如无记录：显示空状态提示
            ↓
    每次进入页面自动刷新数据
```

**入口点**:
- 底部Tab栏"错题本"选项卡
- 图标状态：默认显示 `book.png`，选中显示 `book-active.png`

**页面加载行为**:
- 在 `onShow` 生命周期中加载数据
- 从本地存储读取 `error_history`
- 按时间倒序排列（最新的在前）

### 3.3 错题组织与展示

**列表布局**:

```
┌─────────────────────────────┐
│ 共 N 条记录        [清空全部] │  ← 列表头部
├─────────────────────────────┤
│ ┌─────────────────────────┐ │
│ │ [学科]        [时间]  🗑️▼ │ │  ← 卡片头部（可点击展开）
│ │ 题目摘要内容...         │ │  ← 题目预览
│ └─────────────────────────┘ │
│           ...               │
│ ┌─────────────────────────┐ │
│ │ 数学          2026-04-19│ │  ← 展开状态示例
│ │ 求函数f(x)=x²+2x的导数  │ │
│ ├─────────────────────────┤ │
│ │ 🔍 错因分析             │ │  ← 详情区域
│ │ 学生在求导时忽略了...   │ │
│ ├─────────────────────────┤ │
│ │ 📌 知识点               │ │
│ │ [导数][函数][微积分]    │ │
│ ├─────────────────────────┤ │
│ │ 💡 举一反三             │ │
│ │ 练习题1 [查看答案/收起] │ │
│ │ 练习题2 [查看答案/收起] │ │
│ │ 练习题3 [查看答案/收起] │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

**卡片结构**:
- **头部**: 学科标签（如有）+ 时间戳 + 删除按钮 + 展开箭头
- **预览区**: 题目内容摘要（始终显示）
- **详情区**（展开后显示）:
  - 错因分析
  - 知识点标签列表
  - 举一反三练习题（带答案展开功能）

**排序规则**:
- 按创建时间倒序（最新的记录在前）
- 最多保留100条记录

### 3.4 错题操作

**可用操作**:

| 操作 | 触发方式 | 确认方式 | 效果 |
|------|----------|----------|------|
| 展开详情 | 点击卡片头部 | 无需确认 | 展开显示完整详情，其他收起 |
| 收起详情 | 再次点击头部 | 无需确认 | 收起当前卡片 |
| 删除单条 | 点击 🗑️ 按钮 | 弹窗确认 | 删除该条记录，刷新列表 |
| 清空全部 | 点击"清空全部" | 弹窗确认 | 删除所有记录，显示空状态 |
| 查看答案 | 点击"查看答案" | 无需确认 | 展开练习题答案 |
| 收起答案 | 点击"收起答案" | 无需确认 | 收起练习题答案 |

**删除确认流程**:
```
点击删除按钮
    ↓
显示确认弹窗（wx.showModal）
    ↓
用户点击"确定"
    ↓
执行删除 → 刷新列表 → 显示成功提示
    ↓
用户点击"取消"
    ↓
无操作，返回列表
```

### 3.5 数据流：识别到保存

```
首页拍照识题
    ↓
AI分析完成
    ↓
自动调用 saveToHistory()
    ├─→ 生成记录ID（当前时间戳）
    ├─→ 添加创建时间（中文本地化格式）
    ├─→ 构造记录对象：
    │       {
    │         id: Date.now(),
    │         createTime: "2026/4/19 16:30:45",
    │         imagePath: "...",
    │         original_question: "...",
    │         error_analysis: "...",
    │         subject: "数学",
    │         knowledge_points: [...],
    │         similar_questions: [...]
    │       }
    ├─→ 插入到历史记录数组开头
    ├─→ 限制最多100条（超出时移除最旧的）
    └─→ 保存到本地存储（wx.setStorageSync）
            ↓
    下次进入错题本时读取显示
```

**数据持久化**:
- 存储键名: `error_history`
- 存储方式: `wx.setStorageSync`
- 数据格式: JSON数组
- 容量限制: 最多100条记录

## 4. 设置模块

### 4.1 页面功能概述

**页面**: `pages/settings/settings`
**名称**: 设置

**核心功能**:
1. 配置AI分析API参数
2. 支持默认模式（内置API）和自定义模式
3. 测试API连接
4. 查看使用指南
5. 了解应用信息

**配置项**:
- **默认模式**: 使用内置API（阿里百炼），开箱即用
- **自定义模式**: 配置自己的API URL、API Key、模型名称

### 4.2 用户流程：访问设置

```
点击底部Tab栏【设置】
            ↓
    加载当前配置状态
            ↓
    显示设置页面
        ├─→ API配置区域
        ├─→ 使用指南区域
        └─→ 关于区域
```

**入口点**:
- 底部Tab栏"设置"选项卡
- 图标状态：默认显示 `settings.png`，选中显示 `settings-active.png`

### 4.3 设置选项与效果

#### 4.3.1 API模式选择

| 模式 | 说明 | 配置项 |
|------|------|--------|
| 默认模式 | 使用内置API，无需配置 | 无需填写 |
| 自定义模式 | 使用自己的API服务 | 需填写URL、Key、模型 |

**切换效果**:
- 切换模式时立即生效（自定义模式需保存后才生效）
- 默认模式自动标记为"已配置"
- 自定义模式需验证配置完整性

#### 4.3.2 自定义模式配置项

**API URL**:
- 用途: 大模型API接口地址
- 默认值: `https://open.bigmodel.cn/api/paas/v4/chat/completions`（智谱）
- 支持: 可改为代理或转发地址

**API Key**:
- 用途: API认证密钥
- 安全: 输入框密码显示，可切换明文查看
- 保存后: 显示脱敏形式（如 `sk-xxxx****xxxx`）

**模型名称**:
- 用途: 指定使用的视觉模型
- 示例: `glm-4.6v-flash`、`qwen3.5-plus`
- 要求: 需支持图片理解的多模态模型

**配置保存**:
```
填写完整配置
    ↓
点击【保存配置】
    ↓
验证必填项
    ↓
保存到本地存储
    ├─→ zhipu_api_mode: "custom"
    ├─→ zhipu_api_url: "..."
    ├─→ zhipu_api_key: "..."
    └─→ zhipu_model: "..."
            ↓
    显示保存成功提示
```

**配置清除**:
```
点击【清除配置】（仅自定义模式显示）
    ↓
确认弹窗
    ↓
清除存储的配置值
    ↓
显示已清除提示
    ↓
重置输入框
```

### 4.4 设置持久化机制

**存储键值**:

| 键名 | 说明 | 默认模式 | 自定义模式 |
|------|------|----------|------------|
| `zhipu_api_mode` | 当前模式 | `default` | `custom` |
| `zhipu_api_url` | API地址 | 内置URL | 用户输入 |
| `zhipu_api_key` | API密钥 | 内置Key | 用户输入 |
| `zhipu_model` | 模型名称 | `qwen3.5-plus` | 用户输入 |

**存储方式**:
- API: `wx.setStorageSync` / `wx.getStorageSync`
- 数据持久化到设备本地
- 应用重装后配置丢失（需重新配置）

**配置读取时机**:
- 设置页 `onLoad` 时读取当前配置
- 首页分析前读取API参数
- API调用时使用当前配置

### 4.5 API连接测试

**测试流程**:
```
点击【测试连接】
    ↓
根据当前模式获取配置
    ├─→ 默认模式：使用内置URL和Key
    └─→ 自定义模式：使用用户输入的配置
            ↓
    发送测试请求
            ↓
    显示测试结果
        ├─→ 成功：✅ 连接成功！模型回复：...
        └─→ 失败：❌ 错误信息（认证失败/网络错误等）
```

**测试内容**:
- 发送简单问候语请求
- 验证API可连通性和认证有效性
- 显示模型回复内容（成功时）

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   错题本页    │  │   设置页      │  │  拍照识题页   │
│(pages/history)│  │(pages/settings)│  │  (Tab切换)   │
└──────────────┘  └──────────────┘  └──────────────┘
        │                  │
        │    ┌─────────────┘
        │    │
        ▼    ▼
┌──────────────┐
│   Tab栏切换   │
│ (互相切换)   │
└──────────────┘
```

**导航规则**:
- Tab栏切换：始终可用，点击即切换
- 无深层页面：所有功能都在三个主页面内完成
- 无页面栈：不涉及 `wx.navigateTo`，仅Tab切换

### 5.3 页面间数据传递

**数据传递方式**:

由于采用Tab栏导航，页面间不直接传递数据，而是通过以下方式共享：

1. **本地存储共享** (`wx.getStorageSync` / `wx.setStorageSync`)
   - 错题记录：`error_history` 键
   - API配置：`zhipu_api_*` 系列键

2. **全局数据** (App.globalData)
   - 用户信息：`userInfo`
   - 当前仅用于登录态，未在页面间传递业务数据

**数据流向**:

```
首页（拍照识题）
    │
    ├─→ 分析完成后 ───────────────────┐
    │    saveToHistory()              │
    │         │                       │
    │         ▼                       │
    │    wx.setStorageSync            │
    │   ('error_history', [...])      │
    │                                 │
    └─────────────────────────────────┘
                                      │
    ┌─────────────────────────────────┘
    │
    ▼
错题本页（history）
    │
    └─→ onShow() 时读取
         wx.getStorageSync('error_history')
```

### 5.4 全局应用状态

**全局状态定义**:

| 状态 | 存储位置 | 说明 |
|------|----------|------|
| error_history | localStorage | 错题记录列表 |
| zhipu_api_mode | localStorage | API模式 |
| zhipu_api_url | localStorage | 自定义API地址 |
| zhipu_api_key | localStorage | 自定义API密钥 |
| zhipu_model | localStorage | 自定义模型名称 |
| userInfo | App.globalData | 用户信息（登录态） |

**状态同步机制**:
- 写入：直接调用 `wx.setStorageSync`
- 读取：在页面 `onLoad` 或 `onShow` 时读取
- 更新：错题本页在 `onShow` 时重新加载

### 6.2 完整数据流程

### 6.3 数据限制

| 数据类型 | 限制 | 说明 |
|----------|------|------|
| 错题记录数 | 最多100条 | 超出时自动移除最旧的 |
| 图片大小 | 压缩后上传 | 使用 `sizeType: ['compressed']` |
| 历史记录存储 | 本地存储 | 随小程序删除而清除 |
| API配置存储 | 本地存储 | 需重新配置的情况：重装、清缓存 |
| Token限制 | 4096 | AI分析请求 max_tokens |

## 附录

### A. 页面生命周期

**首页 (index)**:
- `onLoad`: 初始化数据
- 无特殊生命周期处理

**错题本 (history)**:
- `onShow`: 每次显示时重新加载历史记录

**设置 (settings)**:
- `onLoad`: 读取当前配置

### B. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 分析失败 | 显示错误弹窗 (wx.showModal) |
| API认证失败 | 提示检查API Key配置 |
| 网络错误 | 提示检查网络连接 |
| 存储已满 | 自动清理旧记录（保留100条） |

### C. 外部依赖

- **AI服务**: 阿里百炼 (默认) / 智谱GLM (自定义)
- **API端点**: 需在微信公众平台配置为合法域名
- **登录**: 使用微信登录获取用户标识
